# Contributing Guide Generation Guide

This guide helps generate CONTRIBUTING.md files for open source projects.

## When to Use

- Creating a new open source project
- Improving developer experience
- Setting contribution standards

## Key Sections to Include

### 1. Welcome & Introduction

```
# Contributing to [Project Name]

We welcome contributions! This guide explains how to help.
```

### 2. Code of Conduct (Optional)

```
## Code of Be Respectful
```

### 3. How to Contribute

| Way | Description |
|-----|-------------|
| Bug reports | Use GitHub Issues |
| Feature requests | Open a discussion |
| Pull requests | Follow our process |

### 4. Development Setup

```markdown
## Getting Started

1. Fork the repo
2. Clone: `git clone <your-fork>`
3. Install: `npm install`
4. Test: `npm test`
```

### 5. Pull Request Process

1. Create feature branch
2. Make changes
3. Add tests
4. Update docs
5. Submit PR

### 6. Style Guides

- Code style (ESLint, Prettier)
- Commit messages
- Documentation format

### 7. Recognition

```
## Contributors
[List of contributors]
```

## Output Template

```markdown
# Contributing to [Project Name]

## Welcome

## How to Contribute
- Bug reports
- Feature requests  
- Pull requests

## Development Setup
[Setup instructions]

## Pull Request Process
[PR guidelines]

## Style Guide
[Coding standards]

## Questions?
[Contact info]
```

## Tips

- Keep it concise
- Include working code snippets
- Link to external docs
- Add badges for CI/CD status

## Output Location

```
CONTRIBUTING.md
```
